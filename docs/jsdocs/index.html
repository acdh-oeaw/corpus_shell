corpusshell.js

Contains all the helper methods that are called from index.html which do some preliminary checks and then delegate the work to the specialized classes. It also provides the ui logic for the sidebar.
This file has some sections which are just instructions and are not wraped into a function block. Main entry points here are module:corpus_shell~doOnDocumentReady which is passed to jQuery as the $(document).ready() handler, module:corpus_shell~SaveProfileAs, module:corpus_shell~CreateNewProfile, module:corpus_shell~LoadProfile and module:corpus_shell~DeleteProfile which are activated by correponding buttons in the side bar, module:corpus_shell~GetUserData which is used by the Refresh user profiles button and module:corpus_shell~ShowIndexCache which provides the Show/Hide indexes functionality.

panelmanager.js

Provides tha PanelManager class and the global PanelController object.
Entry point from index.html is module:corpus_shell~PanelManager#OpenNewSearchPanel
Entry point from corpusshell.js is module:corpus_shell~PanelManager#OpenSubPanel which is automatically attached to a set of links to replace their default behaviour. See module:corpus_shell~doOnDocumentReady
Currently the target of all operations is the id selector of "#snaptarget" -> module:corpus_shell~PanelController

ressourcemanager.js

Provides a managemant class for search endpoints and indexes module:corpus_shell~ResourceManager and a default object for this purpose module:corpus_shell~ResourceController.
Main entry points are module:corpus_shell~ResourceManager#AddResource and module:corpus_shell~ResourceManager#AddIndex which are called from module:corpus_shell~doOnDocumentReady. module:corpus_shell~ResourceManager#GetIndexCache generates the box that is shown after a click on Show indexes.
