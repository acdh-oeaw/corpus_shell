doOnDocumentReady()

Initialization for the corpus_shell app. Run on $(document).ready().

  1. Bind events to DOM elements that do exist when the document is loaded, that is everything belonging to the side bar.
  2. Bind events to DOM elements that may not exist in the current DOM tree using jQuery. These DOM elements are generated by some search request results so further functionality can be provided.
    • For every link in a .data-view .full class container in a .searchresults the default behavior is replaced by the module:corpus_shell~PanelManager#OpenSubPanel method. This method is used to open a full text sub panel.
    • For every link in a .data-view .full class container in a .searchresults the default behavior is replaced by the module:corpus_shell~PanelManager#OpenSubPanel method. This method is used to open an image sub panel which shows the facsimile.
    • For every link of class .value-caller in a .searchresults for a kwic search result the default behavior is replaced by the module:corpus_shell~PanelManager#OpenSubPanel method.
    • For every link of class .search-caller in a .searchresults for a scan result the default behavior is replaced by the module:corpus_shell~PanelManager#OpenNewSearchPanel method.
    • For every link in a .navigation class container in a .searchresults the default behavior is replaced by the module:corpus_shell~PanelManager#OpenSubPanel method. This method is used to enable page navigation in full text views.
  3. Tries to get a user id.
  4. Initiates asynchrounous loading of the profiles and the index data.
  5. Installs a handler for the module:corpus_shell~PanelManager#event:onUpdated event of module:corpus_shell~PanelController.
  6. Displays a URL which contains the current user ID in a DOM element designated by #userid
  7. Generates a drop down list with search targets using module:corpus_shell~GenerateSearchCombo into the DOM element designated by #searchbuttons and preselects the first item
  8. Generates a drop down list with profiles using module:corpus_shell~GenerateProfileCombo right after the DOM element designated by #profiledel and preselects the first item
Note: The function is written in a style that enables the user/developer to see the program flow until the initialization is finished. This is achieved by deliberately using anonymous functions as callbacks. Please keep in mind that there is a variable delay between the call to Get...() and the execution of the following code block.
